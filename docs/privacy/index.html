Privacy Policy

Plain English. Last updated 2026-04-19.

Short version

NetMon collects no data, runs no analytics, and sends nothing to any third party.

  • All communication happens between your iPhone/iPad and the NetMon Server you run on your own hardware.
  • No telemetry, no crash-reporter-as-a-service, no advertising IDs.
  • No accounts, no sign-in, no email required.

What NetMon touches

Your server URL + API token
Stored in the iOS Keychain on your device. Never synced to iCloud. Used only to talk to your NetMon Server.
Router credentials
Entered on your Mac in the NetMon Server's setup, stored locally on that Mac. The iOS app never sees or handles them.
Network-device data (WAN status, throughput, signal, etc.)
Fetched from your routers by the server, relayed to the iPhone over the network you've configured. This data never leaves your hardware.
Optional push notifications
If you enable push, your device's APNs token is stored on the server you paired with. It's used only by that server, only to push your own alerts to your own device.
Optional InControl 2 cloud integration
If you configure it, the NetMon Server talks directly to Peplink's InControl 2 API using credentials you provide. NetMon itself never brokers or caches those credentials elsewhere.

What NetMon does not do

  • No third-party SDKs. No Firebase, no Mixpanel, no Sentry-as-a-service, no Amplitude, no advertising networks.
  • No background data collection.
  • No sale or sharing of any information, because there isn't any to share.

Children

NetMon is not directed at children under 13. The app has no content aimed at children and does not knowingly collect any data from anyone.

Changes

If this policy ever changes, the update date at the top of this page will reflect it. For a running log of the app, see the public source repository.
